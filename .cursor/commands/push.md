---
description: 変更をコミットして origin へ push する（ワンプッシュ）
---

## 目的

未コミットの変更をステージし、内容に合ったメッセージでコミットしてから `git push` まで行う。

## 手順（必ずこの順で実行）

1. `git status` で変更ファイルを確認する。
2. コミット対象がなければ、その旨を伝えて終了する。
3. ある場合は `git add` で必要なファイルをステージする（通常はリポジトリルートで `git add -A` でよい。意図しないファイルが含まれる場合は除外する）。
4. 差分の要約に基づき、**Conventional Commits** 風の英語プレフィックス（`feat:` / `fix:` / `docs:` / `chore:` など）＋日本語または英語の短い説明でコミットする。例: `docs: 動画ギャラリーに Shorts を追加`
5. `git push` を実行する（現在のブランチの upstream に対して）。
6. 成功・失敗の結果と、使ったコミットメッセージを簡潔に報告する。

## エラー時

- **コンフリクト・rebase 途中**: 自動で直さず、状況を説明しユーザーの判断を仰ぐ。
- **認証エラー・権限不足**: ログやメッセージを要約し、ローカルでの対処（SSH、資格情報、リモート URL）を提案する。

## 注意

- ユーザーが明示したコミットメッセージがある場合はそれを優先する。
- `--force` や履歴改変は、ユーザーが明示的に依頼した場合のみ検討する。
